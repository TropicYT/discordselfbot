# Политика безопасности

## Поддерживаемые версии

Мы выпускаем обновления безопасности только для актуальной версии проекта.
Пожалуйста, убедитесь, что используете последний релиз.

| Версия | Поддерживается          |
| ------ | ----------------------- |
| 2.x.x  | :white_check_mark: Да   |
| 1.x.x  | :x: Нет                 |
| < 1.0  | :x: Нет                 |

## Сообщение об уязвимости

Мы очень серьёзно относимся к безопасности нашего проекта.  
Если вы обнаружили уязвимость, пожалуйста, **не создавайте публичную Issue**.

Вместо этого свяжитесь с нами конфиденциально через один из этих каналов:

- **Email:** [tropic@yumo.su]  
- **GitHub Security Advisory:** используйте вкладку «Security» → «Report a vulnerability» в этом репозитории  
- **Другие контакты:** [https://t.me/tropic_mc]

Пожалуйста, по возможности предоставьте:

- описание уязвимости и её потенциального влияния;
- шаги для воспроизведения (PoC);
- версию проекта, в которой обнаружена проблема.

## Чего ожидать после отправки отчёта

1. **Подтверждение получения** – вы получите ответ в течение [48 часов].
2. **Первичная оценка** – мы проверим отчёт, возможно, запросим дополнительную информацию.
3. **Исправление и раскрытие** –
   - Мы разработаем исправление и выпустим новую версию.
   - После релиза мы опубликуем информацию об уязвимости (в виде CVE/GitHub Advisory) и выразим вам благодарность, если вы этого захотите.

Мы придерживаемся политики скоординированного раскрытия (coordinated disclosure) и просим вас не разглашать детали публично, пока исправление не будет готово.

## Рекомендации по безопасному использованию

- Регулярно обновляйте проект до последней версии.
- Не передавайте секретные ключи и пароли в открытом виде.
- Следуйте примерам из документации для настройки прав доступа.
